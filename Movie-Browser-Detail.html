Movie Browsing Website

Website full stack - 2023

Throughout a six-week development cycle, I created an optimized servlet architecture website for movie browsing through collaboration with my classmate Anthony Wen and guided by Professor Chen Li of Department of Computer Science .

In this project I was able to apply many website servicing techniques taught by the course of COMPSCI 122B, DATA&WEB APPS.

Key implementations

  • Setting up a cloud machine( AWS instance ) to host website, including installation of Tomcat, Java and MySQL.
  • Batch insert of Dataset into MySQl(Movie data, user data, movie stars data.)
  • Applied encryption for account password using Javsypt encryption. Implemented reCAPTCHA authentication to protect spams. Https protocol.
  • Seperated ecosystem design with servlet API hosted on Tomcat and Website(html and javascript) access information through API request.
  • Converted servlets APIs that requires sensitive information to doPost request to avoid inject attacks.
  • Included Dom and Sax parsers to read xml data and insert into the database.
  • kept a session and sticky cookies to allow for seamless interactions between websites and tabs aiming for a better user experience.

Optimization Technique applied

  • Used PreparedStatement to speed up repeating MySQL requests.
  • Optimized Website Traffic through connection pooling, master and slave architecture. Tested with both AWS and Google Cloud, Cloud was faster for this scenario.
  • Implemented fuzzy search by updating table elements with FULLTEXT attributes

Additional features

  • Android App that is a simplified version of the website retaining key features such as login and fuzzy search.

Current Situation

The current set up is linked to old AWS Instances that I deleted to avoid charges from AWS. If needed I can reset up the service and have everything operational.
